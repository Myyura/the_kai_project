#### (2)

The worst case is when the partition is as unbalanced as possible.
Since we want the digits at the start of the array to be as large as possible.
In addition, each integer from 1 to 9 is one each.
Therefore, the worst case is `987654321`, i.e. $X_{\max}$.

For $X_{\min}$, we want this case: (from bottom to top)
```text
       (1,2,3,4,5,6,7,8,9)
                |
        |▔▔▔▔▔▔▔|▔▔▔▔▔▔▔▔▔|
    (1,2,3,4)   5     (6,7,8,9)
        |       |         |
  |▔▔▔▔▔|▔▔▔▔|  |   |▔▔▔▔▔|▔▔▔▔|
(1,2) - 3 - (4) | (6,7) - 8 - (9)
        |       |         |
    (3,4,2,1) - 5 --- (8,9,7,6)
                |
       (5,8,9,7,6,3,4,2,1)
```

Therefore, the best and largest case is `589763421`.

#### (3)

In the worst case, the time complexity $T(n) = T(n-1) + T(0) + O(n)$
Since $T(0)$ is constant, therefore $T(n) = T(n-1) + cn = cn + c(n-1) + c(n-2) + \dots + c(1) \approx c \frac{n(n+1)}{2}$
i.e. $T(n) = O(n^2)$, i.e. the time complexity is $O(n^2)$

#### (4)

A: `swap(X, i, j);`
B: `swap(X, left, i);`
C: `f(X, left, pivotpos-1); f(X, pivotpos+1, right)`

#### (5)

Complexity: $O(n \log n)$
Reason: On average, the randomly chosen pivots splits the array into two subarrays of roughly proportional sizes.
Because balanced or near-balanced splits occur with constant probability, the expected height of the recursion tree becomes logarithmic, i.e. $O(\log n)$.
Since the partitioning operation at each level of recursion trees takes $O(n)$ time in total, the total expected time complexity is $O(n) \cdot O(\log n) = O(n \log n)$.
The expected running time $T(n)$ satisfies: $T(n) = O(n) + \frac{1}{n} \sum_{i=0}^{n-1} (T(i) + T(n-1-i))$
i.e. $T(n) = O(n \log n)$
